- 各リリースは日付付きで記録し、カテゴリ別（Added, Changed, Deprecated, Removed, Fixed, Security）で整理します。

- 廃止・削除された機能（互換性が壊れる変更）。

### Fixed

## 書き方メモ（貢献者向け）

- 変更はまず `Unreleased` セクションに追加してください。
- 変更内容は短く、何をしたかが一目で分かるように記述してください（例: `Added: 新しい CLI コマンド 'foo' を追加`）。
- バグ修正や破壊的変更はそれぞれ `Fixed` / `Removed` に記載してください。
- Pull Request を作成する際、PR 本文にも関連する CHANGELOG の箇所を追記してください。

---

## リリース手順（簡易）

1. `Unreleased` にある項目を確認して整理する。
2. リリースするバージョン番号を決める（セマンティック バージョニングに従う）。
3. `Unreleased` を新しいバージョン見出しに移動し、リリース日を追記する。
4. コミットしてタグを作成する（例: `git tag -a vX.Y.Z -m "Release vX.Y.Z"`）。
5. リリース PR を作成してレビュー・マージする。
